## What is source-ready

- Linux 6.6.50 kernel base.
- Android 16-style `BoardConfigA16.mk`.
- `init_boot`, `vendor_boot`, `vendor_dlkm`, `system_dlkm`, `dtbo.img`.
- A16 fstab with `/vendor_dlkm` and `/system_dlkm`.
- KernelSU/kprobes config.
- NetHunter-friendly networking and USB gadget module base.
- RMX3171 panel/touch/audio/battery/fingerprint/FM slim ports compile and package.
- MTK modem kernel modules compile and package.

## What is not yet production-proven

No physical RMX3171 boot log is present yet. The 6.6 track must remain
"source-ready / test candidate", not "daily-driver proven".

Hardware that still needs real-device validation:

- boot to Android 16 userspace
- display/backlight
- touch events
- speaker/earpiece/mic routing
- battery percent accuracy
- charging current and PE+ negotiation safety
- fingerprint enrollment/unlock
- WiFi/BT firmware load on hardware
- FM/connsys bridge
- modem boot, SIM, calls, LTE data, VoLTE/IMS

Still unsolved in source:

- Mali G52 Bifrost full 3D acceleration
- MTK ISP3 camera
- KMI ABI allowlist
- production AVB keys
- Android clang-r510928/clang-18 production build environment

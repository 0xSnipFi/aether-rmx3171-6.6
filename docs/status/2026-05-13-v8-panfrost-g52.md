## What changed after v7

Panfrost was enabled as the real Linux 6.6 kernel-side path for RMX3171's
Mali-G52 MC2-class.

Build proof:

```text
CONFIG_DRM_PANFROST=m
CONFIG_DRM_SCHED=m
CONFIG_DRM_GEM_SHMEM_HELPER=y
out/drivers/gpu/drm/scheduler/gpu-sched.ko
out/drivers/gpu/drm/panfrost/panfrost.ko
```

DTS proof:

```dts
&mali {
        compatible = "arm,mali-bifrost";
        interrupt-names = "job", "mmu", "gpu", "event";
        status = "okay";
};
```

Why this matters: Samsung's staged MT6768 DTS describes the GPU as
Valhall/Avalon, but RMX3171 is Mali-G52 MC2-class, which is Bifrost. Panfrost can
probe G52 by reading the hardware ID from the GPU block.

## GPU reality

This is not yet a full Android daily-driver GPU claim.

Panfrost gives a real kernel render-node path. Android userspace still needs
one of these:

1. Mesa/Panfrost AOSP graphics stack, including compatible gralloc/HWC path.
2. A matching proprietary Mali Bifrost DDK port from the 4.14 source pool.

Do not mix Panfrost with stock `libGLES_mali.so`; the stock blob expects the
proprietary Mali kernel ABI, not the DRM Panfrost ABI.

It does not compile cleanly yet. Current blockers:

- SMI debug dependency `mtk_smi_dbg_hang_detect()` is not wired for ISP3.
- Full camera still needs ISP3 + imgsensor + sensor tuple + HAL validation.

The staged 6.6 source is useful, but camera is not complete.
